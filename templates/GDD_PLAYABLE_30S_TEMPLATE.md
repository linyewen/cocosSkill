# 30 秒试玩广告策划案模板

> **写作角色分工**：
> - 🧑‍💼 **策划**：写**需求**和**设计意图**（这个游戏是什么、玩家要做什么、想让玩家感到什么）
> - 🤖 **AI（开发）**：根据策划案自动补**数值**（HP / 速度 / 间隔 / 暴击率 / 波次配置）+ **时间轴** + **实现**
> - 🎨 **美术**：提供 **UI 效果图** + **单位碎图**（直接发图不写文字）
> - 🔧 **开发者**：提供 **cocos MCP** + **cocosSkill** + 技术栈（已约定好，策划案不用写）
>
> **填写原则**：
> - 每节开头都写 `（策划填：XXX）`、`（AI 自动：XXX）`、`（美术提供：XXX）` 分清责任
> - 策划不确定的数值留白让 AI 自动设计
> - 有图的地方写"见附件 `ui/xxx.png`" 不写文字描述

## Ch 1 · 核心体验（策划填，3 段话以内）

### 1.1 一句话玩法
> 用动词描述玩家每秒做什么，避免"类似 XX"的模糊说法

```
玩家 <动词 1> 控制 <主角>，<动词 2> 击败 <敌人>，
<动词 3> 成长，最终 <胜利条件>。
```

例子：玩家**左右滑动**控制战机，**自动射击**击败下落的方块，**打道具变强**，最终**击败 Boss**。

### 1.2 情绪曲线（30s 节奏）
> 策划只描述情绪，**具体时间轴由 AI 设计**

```

尾段（约 1/5 时长）：<决战 / 挑战 / 抉择>
```

> 希望玩家脑海里记住的 3 个瞬间

```

3. <另一个时刻>
```

## Ch 4 · 升级成长系统（策划填意图，AI 补数值）

```
升级路线数：<有几条升级路线>
每条意图：
- <路线 1 名字>：<设计目的 - 帮玩家解决什么局面>
- <路线 2 名字>：<设计目的>
- <路线 3 名字>：<设计目的>

升级触发：<打怪 / 捡道具 / 攒货币到阈值>
最大等级：<每条最多升几级>
限时：<一局内能升满 / 永久解锁>
关键约束：<比如"一局内必须给玩家攒到满级才有爽感">
```

例子：
```
升级路线数：3 条
每条意图：
- 多炮：清场速度变快（炮口 1 → 2 → 3 → 4）
- 攻击：单体伤害提高，应对高 HP 方块
- 射速：密集输出，配合多炮成指数爽打

升级触发：打破道具方块（道具方块混在普通方块里，需要玩家主动选择打谁）
最大等级：每条 3 级
限时：一局内满级不持久化
关键约束：必须在前 15 秒内给玩家完成"多炮 → 攻击 → 射速"三件套满级
```

## Ch 5 · 关卡节奏阶段（策划填"想要几个阶段+情绪目标"）

```
阶段数：<建议 3-5 个阶段>
每个阶段：
阶段 1：名字 = <xxx>，目标 = <情绪目标>，主要内容 = <放什么单位 / 关键事件>
阶段 2：...
阶段 N：...

Boss：<有 / 无>，<出现在哪个阶段>
```

例子：
```
阶段数：4 个
阶段 1：教学开场（0-5s）—— 让玩家 2 秒内学会怎么移动，出场 1 个低 HP 方块送玩家爽打
阶段 2：成长期（5-15s）—— 3 波方块，每波穿插 1 个道具方块，让玩家收集 3 种升级
阶段 3：爽打期（15-25s）—— 满级状态下冲一波高 HP 方块墙，给玩家"变强了" 的反馈
阶段 4：决战（25-30s）—— Boss 出现，给 5 秒决一胜负

Boss：有，阶段 4 登场，横跨整排血量超高
```

**⚠️ AI 会自动设计**：
- 每个阶段的具体波次数量
- 每波方块数 / HP 分布 / 道具位置
- 具体时间点（25s、26.5s ...）
- 方块 step-down 速度

策划**不用写**这些数值，只写"**4 个阶段、每个阶段情绪目标**"即可。

## Ch 6 · 反馈系统（策划填"玩家做 X 要感到 Y"）

> 这一节影响"手感"，建议策划认真写。AI 会按这里的描述自动调动画时长 / 震屏强度 / 音效。

| 玩家行为 / 事件 | 期望感受 | 反馈方式（策划写要不要 + 强度描述，AI 自动选参数）|
|---|---|---|
| 子弹命中方块 | 有打击感 | 方块抖动 + 扣血数字跳出 + 轻震屏（强度：小）|
| 暴击（随机） | 爽 | 红色大数字 + 特殊音效（强度：中）|
| 打破道具方块 | 获得感 | 道具图标飞向战机 + 升级光效 + 声音（强度：大）|
| 击杀一个普通方块 | 回馈 | 金币抛物线飞出 |
| 击杀 Boss | 高潮 | 全屏爆炸 + 强震屏 + 胜利音 |
| 方块落地（失败） | 挫败 | 屏幕短暂泛红 + 失败音 |
| 升级 | 变强感 | 战机外观切换 + 子弹外观切换 |

（AI 会自动：具体震屏的像素值 / 动画时长秒数 / 音效选哪个）

## AI 会自动补全的内容（策划不用操心）

收到 GDD 后 AI 会自动设计：
1. **时间轴**（每秒做什么事）— 按 Ch 5 阶段划分 + Ch 1 情绪曲线推导
2. **波次数据**（每波多少方块、HP 分布、道具位置）— 按 Ch 4 升级约束推导
3. **单位数值**（HP / 速度 / 暴击率）— 按目标时长 30s 反推
4. **手感参数**（动画时长 / 震屏强度 / 音效选择）— 按 Ch 6 期望感受推导
5. **升级曲线**（金币阈值）— 按 Ch 5 阶段 + Ch 4 最大等级推导
6. **代码架构**（分层 / 事件 / 对象池）— 按 cocosSkill 规范
7. **prefab 绑定**（@property / UUID / 节点层级）— 按 cocos MCP + cocosSkill/scripts
8. **踩坑规避**（Sprite 颜色乘算 / Button vs TOUCH_END / @property 协调）— 按 cocosSkill memory

## 最终 AI 能交付到什么程度

按这份 GDD 填满后，AI 预期能做到：
- ✅ **功能完整度** 95%（玩法循环、升级、胜负、导流）
- ✅ **UI 还原度** 90%（按效果图精准还原）
- ✅ **节奏感** 80%（按情绪曲线设计，可能需要微调）
- ⚠️ **手感** 70%（最后需要人 1-2 小时调暴击强度、震屏、升级瞬间的反馈）

如果**效果图和碎图不全**，还原度会降到 60-70%（AI 按占位方案做，视觉偏差大）。

如果**时间轴 AI 设计和策划想法不符**，改 `waves.json` 即可，**不用改 GDD**。
