# 从策划案 + 美术资源生成完整游戏

你是一个 Cocos Creator 游戏工厂。输入策划案和美术资源，输出可运行的完整游戏+多个差异化变体。

**严格按以下阶段执行，不跳步，每步完成后向用户确认再进下一步。**

---

## 阶段一：理解（不写代码）

### 1.1 读策划
- 读取用户提供的策划文档（.docx/.md/.txt）
- 提取：核心玩法循环、胜负条件、关卡流程、数值体系
- 如果没有策划文档 → 询问用户口述核心玩法

### 1.2 看效果图/流程图
- 读取所有图片（.png/.jpg），理解视觉风格和交互流程
- 提取：画面风格、UI布局、操作方式、关键帧节点

### 1.3 清点美术资源
- 扫描资源文件夹，按类型分类列出：
  - 角色/敌人（几套？几帧动画？）
  - 子弹/道具
  - UI元素（按钮/图标/背景）
  - 特效/音效
- 评估：哪些是静态引用(textures/)、哪些需要动态加载(resources/)

### 1.4 输出理解文档
向用户确认：
```
核心循环：[一句话描述]
美术资源：[N种角色/N种敌人/N种子弹/N种特效]
预计场景结构：[分几层]
预计prefab：[列出所有需要的prefab]
```

**等用户确认后再进入阶段二。**

---

## 阶段二：设计（不写代码）

### 2.1 场景分层设计
调用 `/cocos-layer` 设计场景层级结构。

### 2.2 Prefab 清单设计
对每个游戏对象，按 `/cocos-create` 流程设计节点结构：
```
XxxPrefab (UITransform + 脚本)
  └── icon (Sprite)
```
列出所有 prefab 的：名称、根节点组件、子节点结构、使用的贴图。

### 2.3 脚本架构设计
- 核心管理器（GameManager）
- 游戏对象脚本（Enemy/Player/Bullet/Coin...）
- 配置数据（GameConfig）
- 事件系统（EventDispatcher）
- 音效管理（SoundManager）

### 2.4 输出设计文档
向用户确认所有 prefab 清单和脚本架构。

**等用户确认后再进入阶段三。**

---

## 阶段三：实现基础版

### 3.0 复用脚本架构审查（复用已有脚本时必做）
如果复用其他项目的脚本，在使用前必须检查：
- 音效管理：是否独立为 SoundManager？不能全堆在 GameManager 里
- 全局状态：是否用单例管理？
- 未使用的 @property：声明了但代码中从未调用的删掉
- 架构分层：是否符合 CLAUDE.md 的组件设计模式？
不符合规范的**先重构再继续**，不要带着技术债往下做。

### 3.1 创建项目基础
- 配置场景（Camera/Canvas/分层节点）
- 创建资源 meta 文件
- 搬运美术资源到正确目录

### 3.2 创建 Prefab
按 `/cocos-create` 流程，用编辑器/MCP创建每个 prefab：
1. 创建节点 → layer = UI_2D
2. 添加 icon 子节点 → Sprite + 贴图 + 颜色
3. 保存为 prefab
4. **在编辑器中确认 prefab 可见且正确**

### 3.3 写脚本
- GameConfig（纯数据）
- EventDispatcher（事件总线）
- GameManager（核心流程）
- 各游戏对象脚本

### 3.4 连线测试
- 场景中绑定所有 @property
- 预览运行基础版
- 确认核心循环跑通：操作→反馈→进度→结局

**向用户演示基础版，确认可玩后再进入阶段四。**

## 阶段四：设计变体（重要！）

### 4.1 变体设计原则
每个变体必须是**2-3种经典机制的组合**，不是单一机制换皮。

三层框架：
| 层级 | 作用 | 节奏 |
|------|------|------|
| 操作层 | 每秒在做什么 | 1-2秒 |
| 策略层 | 每5秒做一个决策 | 5-10秒 |
| 成长层 | 30秒内的变化弧线 | 全程 |

### 4.2 机制库
从以下经典机制中选取2-3个组合：

**操作类**：射击、切割、拖拽、弹弓、挡板、点击放置
**策略类**：资源分配、位置选择、风险博弈、时机判断、路径规划
**成长类**：进化变大、buff叠加、连锁范围递增、分裂倍增、combo延长
**环境类**：可破坏墙壁、重力井、火墙轨迹、反弹墙、陷阱地雷

### 4.3 输出变体策划表
```
| # | 名称 | 操作层 | 策略层 | 成长层 | 视觉差异 |
```

### 4.4 输出操作手册
每个变体写明：操作方式 + 策略要点 + 成长线。

**等用户确认变体方案后再实现。**

---

## 阶段五：实现变体

### 5.1 创建变体所需 prefab
- 新变体需要的可见游戏对象 → 按 `/cocos-create` 流程创建 prefab
- 复用已有 prefab 的优先复用

### 5.2 实现配置系统
- GameVariant.ts：每个变体的配置参数
- 改一个 ID 数字 = 切换游戏

### 5.3 实现模式处理器
- 每个变体一个 ModeHandler
- 严格遵守：
  - 不用 `new Node()` 做可见对象
  - 不用 `require()` / `setTimeout`
  - 坐标用 `worldPosition`
  - 有首帧守卫防止即时胜利/失败

### 5.4 自检清单
每个变体实现后，执行 `/cocos-review` 检查。

---

## 阶段六：测试

### 6.1 逐个变体预览测试
- 改 CURRENT_VARIANT_ID = 0, 1, 2, ... 逐个测试
- 检查：能看见所有游戏对象？操作响应正确？有成长感？

### 6.2 修复并迭代
- 根据用户反馈修复
- 每次修复后重新测试

## 使用方式

用户只需说：

> "这是策划案 [路径]，美术资源在 [路径]，帮我生成游戏"

或者：

> "我有一个 [类型] 游戏的想法：[一句话描述]，素材在 [路径]"

AI 自动启动以上6个阶段。
